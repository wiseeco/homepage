IRIS 소개

빌드는 소스 코드를 컴파일이나 관련 라이브러리의 링크를 통해 최종 컴파일러가 인식할 수 있는 실행 파일을 만드는 것을 말하고 배포는 어플리케이션이나 웹 서비스의 형태로 고객이 사용가능한 상태를 만들기 위해서 빌드한 결과물을 서비스하는 서버나 PC에 전달하고 구동하는 등의 작업을 행하는 것이 배포이다. 당사(와이즈에코)에서 개발한 제품인 IRIS는 앞에서 설명한 빌드와 배포를 자동으로 해 주는데 필요한 기능들을 추가해서 개발자에게는 개발에만 전념할 시간을 만들어주고 기업에는 품질 좋은 제품과 개발자의 잉여력을 높여 기업의 가치를 높이는 데 일조하는 것을 목표로 하고 있다.

IRIS 아키텍처

빌드와 배포를 위한 기본 Feature로 5가지 메인 기능(권한관리, 설정, 빌드, 배포, 통계) 으로 개발되어 있다.
jenkins가 필요한 빌드를 수행, chef가 필요한 소프트웨어를 설치, glu가 배포를 진행한다. 그리고 예약 작업은 Chronos를 통해서, 소스 관리는 gitlab, gitblit 등 오픈 소스를 통해 제공, 산출물 관리는 artifactory에서 한다. 롤 부분은 테스트 커버리지, 코드 커버리지, 정적 분석 지표, 관리자 등을 선택해서 배포 권한을 줄 수 있다.


IRIS 실행 시나리오

  1. 먼저 git 서버에서 리포지토리 정보를 가져와서 사용자가 볼 수 있는 프로젝트 리스트를 보여주고 등록할 프로젝트를 선택한다.
  2. 빌드 서버를 등록하고, 배포할 서버를 등록한 다음, 등록할 프로젝트에 빌드와 배포 설정 정보를 셋팅해서 빌드와 배포를 진행할 수 있도록 한다. 필요할 경우 스케쥴링 기능이 있으면 자동으로 빌드와 배포를 수행하게 할 수 있다.
  3. 빌드 버튼을 클릭해서 빌드를 수행한다. 이 때 서버는 빌드를 수행하고 빌드 결과를 DB에 기록하여 해당 프로젝트의 통계 정보와 관리자가 전체 프로젝트의 상황을 알 수 있게 된다.
  4. 빌드가 정상적으로 끝났을 경우 빌드 산출믈을 리포지토리에 저장한다.
  5. 그 산출물을 기록에 남겨서 배포 시 선택할 수 있도록 한다.
  6. 사용자가 배포 버튼을 클릭하면 빌드의 최종 결과물 리스트를 시간순으로 보여줘 배포에 사용될 산출물을 선택하게 한다. 그리고 선택해 배포 버튼을 클릭하면 기존에 설정단계에서 등록된 서버로 빌드 산출물을 전송한다. 이때 결과 로그를 기록하고 진행 상태를 표시하며 배포가 실패했을 경우 롤백 버튼을 활성화해서 롤백을 빠르게 수행할 수 있도록 한다.
  7. 배포가 정상적으로 끝나면 통계 페이지에서 결과를 그래프 형태로 보여줘서 결과를 쉽게 인지 할 수 있도록 한다.
  8. 데모 : http://14.63.197.162/

WISEPUSH 소개

  • MQTT 스펙을 준수한 모바일과 웹상에서 실시간 푸시 메세지를 전송해 주는 제품 (현재 웹과 아이폰(SDK 개발중), 안드로이드를 지원)
  • MQTT 스펙을 준수한 PUB/SUB 모델을 기반으로 토픽이라는 채널을 통해 메세지를 주고 받는 구조로 구현
  • 제한된 컴퓨팅 성능과 빈약한 네트워크 연결 환경에 잘 맞는 메시징 프로토콜인 MQTT를 사용해 다양한 분야에 적용이 가능
  • 데모 : http://www.wiseeco.com:8080/pusher/

WISEPUSH 장점

  • MQTT 기반이어서 모바일에서 저전력으로 구동되어 경량의 푸시 시스템을 실현이 가능
  • M2M 등 다양한 분야로 확산이 용이
  • GCM이나 APNS 연동에서 처럼 사전 등록 과정의 진입장벽을 해소되고, 푸시 제약 정책이 없어짐
  • 푸시 메세지의 END-POINT까지 딜리버리 보증이 가능
  • OASIS(Organization for the Advancement of Structured Information Standards)가 사물 인터넷(Internet of Thing)을 위한 메시징 프로토콜로 MQTT(Message Queuing Telemetry Transport)를 선정했다고 공표해 저희 wisepush도 향후 M2M 분야도 지원을 강화해 나갈 예정이며 Push Notification API 시스템도 곧 도입할 예정
